backend/controllers/accountController.js
----------------------------------------
File: `backend/controllers/accountController.js`
1. Query:
   `SELECT branch_id FROM Accounts WHERE customer_id = ? ORDER BY opened_date DESC, account_id DESC LIMIT 1`
   Used in: `getBranchIdForCustomer`
   Purpose: Finds the branch of the customer's latest account so new accounts are opened in the same branch.

2. Query:
   `SELECT branch_id FROM Branches ORDER BY branch_id ASC LIMIT 1`
   Used in: `getBranchIdForCustomer`
   Purpose: Fallback branch lookup when the customer has no existing accounts.

3. Query:
   `SELECT account_id, account_number, account_type, account_balance, account_currency, account_status, opened_date, closed_date, annual_interest_rate, branch_id, customer_id FROM Accounts WHERE account_id = ? AND customer_id = ? LIMIT 1`
   Used in: `getOwnedAccount`
   Purpose: Confirms the account belongs to the logged-in customer and loads its status/details.

4. Query:
   `SELECT * FROM Accounts`
   Used in: `getAccounts`
   Purpose: Returns all accounts.

5. Query:
   `SELECT * FROM Accounts WHERE account_id = ?`
   Used in: `getAccountById`
   Purpose: Returns a single account by id.

6. Query:
   `SELECT a.account_id, a.account_type, a.account_balance, a.account_status, c.customer_id, CONCAT(c.first_name, ' ', c.last_name) AS customer_name, c.customer_email FROM Accounts a JOIN Customers c ON a.customer_id = c.customer_id WHERE a.account_id = ?`
   Used in: `getAccountDetails`
   Purpose: Returns account details together with customer information.

7. Query:
   `INSERT INTO Accounts (account_number, customer_id, branch_id, account_type, account_balance, account_currency, account_status, opened_date) VALUES (?, ?, ?, ?, 0, 'INR', 'Active', CURDATE())`
   Used in: `createMyAccount`
   Purpose: Creates a new customer account with zero opening balance.

8. Query:
   `SELECT a.account_id, a.account_number, a.account_type, a.account_balance, a.account_currency, a.account_status, a.opened_date, a.closed_date, a.annual_interest_rate, b.branch_name, b.branch_city, b.branch_state, b.ifsc_code FROM Accounts a JOIN Branches b ON b.branch_id = a.branch_id WHERE a.account_id = ? LIMIT 1`
   Used in: `createMyAccount`, `updateMyAccountStatus`
   Purpose: Loads the newly created or newly updated account with branch metadata for the API response.

9. Query:
   `UPDATE Accounts SET account_status = ?, closed_date = CURDATE()|NULL WHERE account_id = ? AND customer_id = ?`
   Used in: `updateMyAccountStatus`
   Purpose: Changes customer-controlled account status and sets or clears the close date.

Notes
-----
1. `optionalQuery(...)` in `customerController.js` and `adminController.js` is a helper wrapper around normal SQL queries. It returns a fallback value when a table does not exist, mainly for optional modules like `Cards` or `Loan_Applications`.
2. Queries using `${placeholders}` are dynamically built `IN (...)` queries in the admin dashboard for pending-loan review context.
3. Queries using `${tableName}` and `${columnName}` are dynamic reference-number uniqueness checks, not user-facing business queries.
